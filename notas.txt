Completando la base de datos

-En esta clase vemos los metodos faltantes del store
-Cuando en un query colocamos ?, esto hace referencia que, como 2do parametro del query ya
  no vendrá el callback, sino el valor de ese ?, despues sí vendrá el callback.
-Sí son varios, se les da valor en el 2do parametro igual, pero en un array.

get:
  -Tendrá un id por parametro.
  -Su query sería:
    .SELECT * FROM [tabla] WHERE id=[id]

insert:
  -Esta será una de las 2 funciones que pueden ser ejecutadas dentro de upsert.
  -Query:
    .INSERT INTO ${table} SET ?
    .El '?' será la data.

update:
  -Esta será la otra que puede ejecutar el upsert.
  -Query:
    .UPDATE ${table} SET ? WHERE id=?
    .El 1er ? será la data, el 2do el id que vendrá en esa data.

upsert:
  -Esta tendrá un 3er parametro, isNew.
  -Con el definiremos sí usar insert o update, dependiendo sí es o no nuevo.

query:
  -Para obtener el token.
  -Query:
    .SELECT * FROM ${table} WHERE ?
    .? será el queryObject (2do param).
  -Esta consulta retorna un array, de la cual, necesitamos solo la 1er posición.
