Documentación de nuestra API

-Es buena practica documentar nuestra api con todo lo que hará.
-Para eso usaremos swagger: https://editor.swagger.io/
-Es mejor ver su documentación para saber las opciones que tenemos.
-Cuando terminemos de crear la documentación, en File > Convert and save as JSON, nos guarda
  un .json en donde queramos con esa documentación. Lo moveremos en la carpeta 'api'.
-Para poder mostrarla en una ruta de nuestra página, iremos al index, la servimos en una 
  ruta con use(), como 2do parametro, usaremos el elemento serve, de una dependencia que 
  tenemos que instalar, llamada 'swagger-ui-express', como 3er param. usamos tambien su 
  metodo setup(), y como parametro, el .json descargado.

-Metodos del index:
  -En el network, podemos notar cosas diferentes que podemos hacer:
    .Podemos tener varios metodos del mismo tipo, en donde usemos un 'params' para cambiar
      la ruta, pero que sea el mismo metodo.
    .El mismo metodo del controller puede usarse en distintos metodos del router, tambien 
      se pueden usar distintos parametros para ellos.
    .Como parametro de algun metodo del parametro, podemos pasar por ejemplo el body, 
      haciendo que en el controller manipulemos lo que queramos de ese body.

nanoid:
  -nanoid es una dependencia de las más usadas para generar ids.
  -Cuando vayamos a dar un valor, usamos esta dependencia, como un metodo, lo damos como
    valor.
  -Regularmente esto lo hacemos en el controller.
