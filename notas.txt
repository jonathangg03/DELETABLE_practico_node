Desplegando los servicios de Node

-Ya tenemos la API corriendo en serverless, pero por algunas razones podríamos quererlo tener corriendo en una VM.
-Como ejemplo, que algunos servicios no corran, o que necesitemos alguna conf. especifica.

-Iremos a Google Cloud Platform, a la sección de Compute Engine > Instancias de VM.
-Crearemos una instancia.
-Al crearla, nos vamos al final, donde dice SSH, y damos click, nos abrirá una consola.

En consola:
  -Nos aseguramos que estemos en el home (cd ~).
  -Descargamos e instalamos los paquetes de node:
    .curl -sL https://deb.nodesource.com/setup_12.x -o nodesource_setup.sh: Nos descarga un script.
    .sudo bash nodesource_setup.sh: Corremos ese script.
  -Instalamos nodejs: sudo apt-get install -y nodejs
  -Instalamos git: sudo apt-get install git.
  -Instalamos pm2 global con npm.
  -Clonamos nuestro repositorio de GH.
  -Ingresamos a la carpeta clonada, e instalamos las dependencias.
  -Añadimos todos los servicios a pm2.

-En este clase ya tenemos la api corriendo en la VM, en cada puerto, pero no lo podemos exponer todavía a internet,
  porque no queremos exponer cada puerto con su ruta, sino que queremos exponer ciertos puertos, pero que vengan
  directamente en la url que tengamos, en este caso la IP del servidor, pero sí tuvieramos el dominio, pues sería
  ese. La mejor forma de hacer esto es con un proxy por delante, llamado NGINX.
