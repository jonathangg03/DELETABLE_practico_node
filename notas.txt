Conectar la API al caché

-Ya tenemos creados los metodos para almacenar la cache en una DB.
-En esta clase veremos como conectar ese servicio a la API.
-Antes, como correremos esto en terminal, debemos fijarnos que en la confesten como antes las variables de entorno.
-Ahorá, para conectar, afectaremos el index del user, y el controller, en los metodos de get, que es donde usaremos
  estos (el get queda como reto, por ahora haremos el list).

index:
  -Haremos exactamente lo mismo que con el store, creamos una variable para el cache, donde sí el remoteDb de la
    conf es verdadero, con la variable importamos a remoteMyCache, de lo contrario, importamos a redis.
  -Al exportar el controller, colocaremos ese cache como 2do parametro.

controller:
  -Como pasamos el cache como 2do parametro, dentro del controller realizaremos la misma validación que con el 
    store, donde en caso de que no se coloque se usará dummy.
  -NOTA: Por ahora usamos dummy, pero sí enviamos a producción, aquí debemos colocar una DB que nosotros tengamos.
  -El metodo list() lo modificaremos para que haga lo siguiente:
    .Traemos el metodo list de cache() a una variable.
    .Sí esta no tiene respuesta, usamos el list del store,y eso lo retornaremos.
    -En caso de que sí de respuesta, los datos traídos con el cache son los que retornaremos.
  -Para ver el funcionamiento, logueamos algo en caso de que se trayeran datos de store o de cache, para ver cuando 
    se hizo una u otra.

-Debemos realizar el reto del get.
