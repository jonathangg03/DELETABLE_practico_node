Gestión avanzada de errores: Throw

-En muchas partes del código estamos 'Lanzando' errores.
-Al hacerlo, no estamos escogiendo un código para cada uno, sino que estamos
  colocando el mismo para toodos.
-Para ello, crearemos en el network de la raíz un archivo 'errors', y otro 'error'
  pero en la carpeta 'utils' de la raíz.

errors:
  -Traemos response.
  -Exportamos una función con los params: error, req, res.
  -Hacemos el log al inicio sí se desea.
  -Usamos el metodo de error de response, y damos valor, el mensaje y el estatus
    lo sacamos del error del 1er param.
  -Este elemento lo importamos en el index de la app, y lo servimos sin ruta, pero
    al final de todas las rutas.

error:
  -Exportamos una función que tenga como params el mensaje y codigo de error.
  -Retornamos una const, que tenga una instancia de la API Error de JS, y tenga
    por parametro el mensaje.
  -Le daremos a esa const, a su elemento code el codigo sí viene uno. 
  -Este lo traemos en donde hayamos 'lanzado' errores, en lugar de lanzar 
    'new Error', lanzamos este metodo, con el mensaje y el código deseados.
